# SonoForge

### Открытая платформа для анализа эхокардиографии

**SonoForge** — бесплатное открытое десктопное приложение для **анализа эхокардиографических исследований**, **просмотра DICOM**, **кардиологических измерений** и **клинической отчётности**. Создано для кардиологов, сонографов и исследователей, которым нужен мощный офлайн-инструмент, соответствующий **рекомендациям ASE (Американское общество эхокардиографии)**.

## 📦 Установка

<details open>
<summary><strong>Linux (.deb) — Рекомендуется</strong></summary>

```bash
# Скачать последний релиз
wget https://github.com/areatu/SonoForge/releases/latest/download/sonoforge_*.deb

# Установить
sudo dpkg -i sonoforge_*.deb

# Запустить
sonoforge
```

При первом запуске автоматически создаётся виртуальное окружение, устанавливаются зависимости и (опционально) скачиваются AI-модели для сегментации.

</details>

<details>
<summary><strong>Windows (.zip)</strong></summary>

1. Скачайте `SonoForge-*.zip` из [Releases](https://github.com/areatu/SonoForge/releases)
2. Распакуйте в любую папку
3. Запустите `SonoForge\bin\SonoForge.bat`

> **Требуется:** Python 3.10+ ([скачать](https://www.python.org/downloads/), поставить галочку "Add to PATH" при установке)

При первом запуске автоматически настраивается окружение и устанавливаются зависимости.

</details>

<details>
<summary><strong>Из исходников (для разработчиков)</strong></summary>

```bash
git clone https://github.com/areatu/SonoForge.git
cd SonoForge

# С uv (рекомендуется)
uv sync --extra dev
uv run sonoforge

# Или pip
pip install -e ".[dev]"
python -m echo_personal_tool
```

</details>

## 🚀 Возможности

SonoForge предоставляет комплексный набор инструментов для **эхокардиографической оценки** — от базовых измерений до продвинутого AI-анализа.

### 📊 Кардиологические измерения

| Категория | Измерения | Описание |
|-----------|-----------|----------|
| **Линейные (M-Mode/B-Mode)** | КДР, КСР, МЖП, ЗСЛЖ, ТАПСЕ, ПЖЖ | Стандартные линейные измерения ASE с подписями в реальном времени |
| **Объёмные (Simpson биплан)** | ОДВ, ОСВ, ФВЛЖ, ОПЛЗ, ОППЗ | Метод Симпсона с отслеживанием митрального кольца |
| **Допплер (спектральный)** | Митральный поток (E, A, E/A), ТВС, СВС, пиковые скорости | Полный допплер-анализ с автоматическим детектированием пиков |
| **Тканевый допплер (TDI)** | e', a', s', E/e' | Оценка диастолической функции по ASE |
| **M-Mode** | Толщина задней стенки, размеры ЛЖ, ФС | Измерения времени-глубины с оверлеем scan line |
| **Стресс (STE)** | GLS, сегментарная деформация, карта AHA 17 сегментов | Спекл-трекинг с NCC block-matching |
| **Функция ПЖ** | ФАП (фракционная площадь изменения), ТАПСЕ | Оценка правого желудочка |
| **Диастолическая функция** | E/A, E/e', скорость ТК, объём ЛЗ | Градация диастолической дисфункции (I-III степень) |
| **Масса ЛЖ** | МЛЖ, ЛМИ (индексация по ВТС), ОСС | Геометрический и анатомический расчёт массы |
| **Площадь тела** | Формула Дюбуа, индексированные измерения | Автоматическая индексация по ВТС |

### 🤖 AI-сегментация

SonoForge интегрирует **ONNX Runtime** для сегментации сердечных структур в реальном времени:

- **Автосегментация ЛЖ** — Автоматическое контурирование левого желудочка в проекции A4C с использованием модели EchoNet-Dynamic (клавиша `I`)
- **Сегментация ЛП** — Контурирование полости левого предсердия в конечной систоле
- **Детекция митрального кольца** — AI-определение landmarks митрального клапана
- **Темпоральная фузия** — Многокадровая согласованность через голосование N±2 соседних кадров
- **Рефайнмент контура** — Edge-snapping и градиентная доработка контура (клавиша `R`)
- **Open-Arc Simpson** — Ручная инициализация контура с точками митрального кольца и апексом

### 🏥 Интеграция с DICOM и PACS

Полная DICOM-совместимость для бесшовной интеграции с больничными информационными системами:

| Протокол | Операции | Описание |
|----------|----------|----------|
| **DICOMweb (WADO-RS)** | QIDO-RS, WADO-RS, STOW-RS | HTTP-доступ к DICOM (по умолчанию) |
| **DIMSE (C-FIND)** | Поиск Study/Series/Instance | Запрос исследований в PACS |
| **DIMSE (C-GET)** | Загрузка отдельных объектов | Скачивание DICOM через DIMSE |
| **DIMSE (C-MOVE)** | Массовая загрузка серий | Перемещение объектов в embedded Storage SCP |
| **DIMSE (C-STORE)** | Загрузка DICOM | Отправка локальных DICOM-файлов в PACS |
| **TLS** | Защищённые ассоциации | Безопасная DIMSE-связь с валидацией сертификатов |

**Поддерживаемые PACS:** Orthanc, DCM4CHEE, Conquest и любой сервер, совместимый с DICOMweb/DIMSE.

### 📈 Клиническая отчётность и экспорт

- **Сводка по исследованию** — Комплексный отчёт со всеми измерениями, расчётами и индексированными значениями
- **Экспорт в PDF** — Клинические отчёты с информацией пациента, измерениями и референсными диапазонами
- **Референсные нормы ASE** — Встроенные таблицы нормативов для взрослой эхокардиологии (по полу/возрасту)
- **Структурированные отчёты** — DICOM SR-совместимый вывод
- **Конструктор справочника** — Редактор с импортом Excel, экспортом в PDF/HTML

### 🎨 Интерфейс и UX

- **Тёмная/светлая тема** — Клинические цветовые схемы, оптимизированные для длительной работы
- **Дуальный просмотр** — Сравнение двух фаз или модальностей бок о бок
- **Галерея** — Навигация по исследованиям через thumbnails
- **Cine-воспроизведение** — Плавный воспроизведение циклов с регулируемой скоростью
- **Окно/уровень** — Интерактивная настройка контрастности/яркости
- **Центровка** — Пространственная привязка между синхронизированными видами
- **Горячие клавиши** — Полная клавиатурная навигация для эффективной работы

## 🏃 Быстрый старт

### 1. Открытие DICOM-данных

- **Локальная папка:** Файл → Открыть папку → Выбрать директорию с DICOM/MP4/JPEG
- **Сервер PACS:** Файл → Загрузить с сервера → Выбрать сервер Orthanc/DICOMweb

### 2. Навигация по исследованиям

- **Галерея** → Выбрать серию → Кадр открывается в основном просмотрщике
- **Прокрутка** через кадры cine с помощью колёсика мыши или стрелок
- **Play/Pause** клавишей `Space` для автоматического цикла

### 3. Выполнение измерений

| Инструмент | Клавиша | Описание |
|------------|---------|----------|
| Линейный калипер | `L` | Измерение расстояния (КДР, МЖП, ТАПСЕ и др.) |
| Simpson биплан | `C` | Измерение объёма ЛЖ (open-arc контур) |
| Допплер пик | `M` | Пиковая скорость спектрального допплера |
| Допплер интервал | `T` | Измерение временного интервала |
| Трассировка ТВС | `V` | Трассировка velocity-time integral |
| M-Mode | `M` | Трассировка и измерения M-Mode |
| Спекл-трекинг | `STE` | GLS и анализ деформации |

### 4. Просмотр результатов

- **Панель результатов** — Все измерения с индексированными значениями (ВТС, поправки на пол/возраст)
- **Экспорт в PDF** — Генерация клинического отчёта
- **DICOM SR** — Сохранение структурированного отчёта в PACS

## 🏗️ Архитектура

SonoForge построен по принципам **Clean Architecture** с чётким разделением ответственности:

```
src/echo_personal_tool/
├── domain/              # Бизнес-логика (без зависимости от Qt)
│   ├── models/          # Модели: Contour, Doppler, Speckle, MMode
│   ├── calculations/    # Расчёты: Simpson, Bernoulli, Teichholz
│   └── services/        # Сегментация, трекинг, референсные данные
├── infrastructure/      # Внешние интеграции
│   ├── dicom_*.py       # Чтение/запись DICOM (pydicom)
│   ├── orthanc_*.py     # DICOMweb клиент (httpx)
│   ├── dimse_*.py       # DIMSE клиент (pynetdicom)
│   ├── onnx_engine.py   # ONNX инференс
│   └── server_settings.py # Управление подключениями
├── application/         # Слой оркестрации
│   ├── app_controller.py # Главный контроллер приложения
│   ├── workers/         # Фоновые задачи (11 параллельных)
│   └── services/        # Сервисы приложения
├── presentation/        # GUI (PySide6/Qt)
│   ├── main_window.py   # Главное окно
│   ├── viewer_widget.py # Просмотрщик DICOM
│   ├── doppler_widget.py # Отображение допплера
│   └── ...              # 30+ UI-компонентов
├── constructor/         # Редактор справочника
└── resources/           # Шрифты, иконки, данные ASE
```

## 🛡️ Безопасность и конфиденциальность

> **Ваши данные остаются локально.** SonoForge обрабатывает все DICOM-данные в оперативной памяти — PHI (защищённая медицинская информация) не записывается на диск, нет загрузок в облако, нет телеметрии.

### Функции безопасности

- ✅ **Валидация DICOM** — Проверка целостности файла перед парсингом (magic bytes, лимиты размера)
- ✅ **Целостность моделей** — SHA256-проверка ONNX-моделей при загрузке
- ✅ **Сетевые таймауты** — Настраиваемые таймауты для DICOMweb/DIMSE
- ✅ **Санитизация PHI** — Обрезка идентификаторов пациентов в логах
- ✅ **Обработка в памяти** — Все DICOM-данные обрабатываются в RAM, без temp-файлов
- ✅ **Офлайн-режим** — Полностью работает без интернета после установки

Подробности: [SECURITY.md](SECURITY.md).
