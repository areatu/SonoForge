Механизм объединяет B-сплайны (гарантирующие математическую гладкость и локальную управляемость кривой) с весовыми функциями расстояния (RBF/IDW) для интуитивной деформации. При движении мыши каждой контрольной точке динамически присваивается вес по формуле (например, гауссовой): чем ближе точка к курсору, тем сильнее она смещается вслед за ним, а удаленные точки получают вес, близкий к нулю. Это создает эффект «мягкого магнита»: количество вовлеченных в коррекцию точек (1, 3 или 5) определяется исключительно радиусом влияния функции, что позволяет локально править контур без появления углов, разрывов и необходимости ручного выделения вершин.

Контур представляется как B-сплайн — гладкая параметрическая кривая, управляемая дискретным набором контрольных точек. При движении курсора каждая точка смещается не одинаково, а пропорционально весу, рассчитанному функцией IDW или гауссовской RBF от евклидова расстояния до курсора. Это создаёт эффект «мягкой кисти»: влияние плавно затухает с удалением, автоматически масштабируя зону редактирования (от 1 до N точек) без потери глобальной гладкости. В результате курсор выступает интуитивным якором деформации, а математическая модель сплайна гарантирует отсутствие изломов и артефактов при любой степени локальной коррекции.

# Техническая спецификация: Интерактивная коррекция B-сплайнового контура (Python)

## 1. Стек технологий
*   **Язык:** Python 3.10+
*   **Численные вычисления:** `numpy` (строго векторизованные операции, без циклов `for` для расчетов расстояний и весов).
*   **Интерполяция:** `scipy.interpolate` (`splprep`, `splev`).
*   **GUI/Визуализация (прототип):** `matplotlib` (использовать `mpl_connect` для событий мыши) или `PyQt`/`PySide` (для продакшена).

## 2. Математическая модель данных
*   **Контрольные точки (Control Points):** Массив `numpy` формы `(N, 2)`, где $N$ — количество точек, $2$ — координаты $(x, y)$.
*   **B-сплайн:** Параметрическая кривая $C(u) = (x(u), y(u))$ для $u \in [0, 1]$, аппроксимирующая или интерполирующая контрольные точки.
*   **Состояние сплайна:** Кортеж `tck` (knots, coefficients, degree), возвращаемый `splprep`.

## 3. Алгоритм взвешенного смещения (Ядро логики)
При событии движения мыши (`on_mouse_move`) при зажатой кнопке, для каждой контрольной точки $P_i$ выполняется следующее:

1.  **Вектор смещения курсора:**
    $\Delta C = C_{current} - C_{previous}$
2.  **Расстояние до курсора (евклидово, векторизованно):**
    $d_i = ||P_i - C_{current}||_2$ для всех $i \in [0, N-1]$.
3.  **Расчет весов (выбрать одну из двух функций):**
    *   *Вариант А (Рекомендуемый, Gaussian RBF):*
        $w_i = \exp\left(-\frac{d_i^2}{2\sigma^2}\right)$
    *   *Вариант Б (Inverse Distance Weighting, IDW):*
        $w_i = \frac{1}{(d_i + \epsilon)^p}$
4.  **Применение смещения:**
    $P_i^{new} = P_i + (\Delta C \times w_i \times K)$
    *(где $K$ — коэффициент чувствительности/усиления, обычно $1.0 \le K \le 2.0$)*.

## 4. Пайплайн обработки событий (Event Loop)
Реализовать в виде ООП-класса (например, `InteractiveSplineEditor`):

*   **`__init__`**: Инициализация массива `control_points`, параметров весовой функции ($\sigma$, $K$), флагов состояния мыши (`is_dragging = False`, `last_cursor_pos = None`).
*   **`on_press`**:
    *   Проверить попадание в область графика.
    *   `is_dragging = True`
    *   `last_cursor_pos = [event.xdata, event.ydata]`
*   **`on_motion`**:
    *   Если не `is_dragging` или `last_cursor_pos is None` → `return`.
    *   `current_pos = [event.xdata, event.ydata]`
    *   `delta = current_pos - last_cursor_pos`
    *   Если `norm(delta) < 1e-3` → `return` (защита от микродрожания).
    *   Вычислить массив расстояний $d$ (векторизованно через `np.linalg.norm`).
    *   Вычислить массив весов $w$ (выбранная функция).
    *   Обновить `control_points`: `self.control_points += delta * w * K` (использовать broadcasting numpy).
    *   `last_cursor_pos = current_pos`
    *   Вызвать метод пересчета и отрисовки сплайна.
*   **`on_release`**:
    *   `is_dragging = False`
    *   `last_cursor_pos = None`
*   **`update_spline`**:
    *   `tck, u = splprep([self.control_points[:, 0], self.control_points[:, 1]], s=0, per=False)`
    *   `u_fine = np.linspace(0, 1, 200)`
    *   `x_spline, y_spline = splev(u_fine, tck)`
    *   Обновить данные линий в GUI и вызвать `canvas.draw()`.

## 5. Гиперпараметры по умолчанию (для инициализации)
```python
WEIGHT_METHOD = 'gaussian'  # или 'idw'
SIGMA = 30.0                # Ширина гауссиана (в пикселях координат). Регулирует "радиус" влияния (1-3-5 точек).
EPSILON = 1e-8              # Защита от деления на ноль для IDW.
P_POWER = 2.0               # Степень затухания для IDW.
SENSITIVITY_K = 1.5         # Множитель силы смещения.
SPLINE_SMOOTHING = 0.0      # Параметр `s` в splprep. 0.0 = строгая интерполяция точек.
```

## 6. Требования к генерируемому коду
1.  **Векторизация:** Запрещено использовать циклы `for` для расчета расстояний и весов. Использовать возможности `numpy` (broadcasting).
2.  **Инкапсуляция:** Логика расчета весов и смещения должна быть вынесена в отдельный метод `_apply_weighted_displacement`.
3.  **Визуальная обратная связь:** Метод отрисовки должен визуально выделять (например, зеленым цветом) те контрольные точки, чей вес $w_i > 0.1$ (порог "активности").
4.  **Безопасность:** Обработка случаев, когда `event.xdata` или `event.ydata` равны `None` (курсор за пределами осей).
